【背景】
PC智能任务助理（二版）和时间切片系统（二版）已完成，均为独立Flask + Edge --app桌面应用。
现在要将两个系统合并为一个统一的原生桌面应用，类似 Llama.cpp-Desktop.exe —— 单一exe，双击即用。

当前代码位置：
- First CC/PC智能任务助理/  (Flask :5000)
- First CC/时间切片系统/    (Flask :5001)

第一阶段：Flask 合并（解决P0路由重复）
────────────────────────────

1. 新建 app.py —— 统一Flask应用
   - 所有路由在此定义，用 url_prefix 区分：
     /api/tasks/*    → PC助理API（任务CRUD、时段CRUD、排程、分配）
     /api/slices/*   → 切片API（切片CRUD、设备日志查询、热力图数据、关联任务列表）
     /api/system/*   → 系统API（通知设置、数据导出、日志清理）
     /               → 主页面 (index.html)
   - 从原 app.py（PC助理）和原 app.py（切片）各提取路由，不复写、不重复
   - 数据库文件路径：data/task_assistant.db 和 data/slices.db
   - 端口：5000（统一）

2. 新建 database.py —— 统一连接管理器
   - 管理两个独立的 SQLite 连接
   - 跨库查询：slices 侧需要读 tasks 时，ATTACH data/task_assistant.db
   - 路径使用相对路径：os.path.join(os.path.dirname(__file__), 'data', 'xxx.db')
   - 保留原有所有 CRUD 函数，按模块分段注释
   - WAL 模式两个库都开启

3. 改造 desktop.py
   - 导入 app.py 的 Flask 实例（from app import app），不在 desktop.py 内定义路由
   - Flask 在后台线程启动
   - Edge --app 模式打开 http://127.0.0.1:5000
   - 系统托盘：显示/退出 + 首次关闭提示气泡
   - 日志写入 logs/ 目录，超过7天自动清理
   - 退出时停止 monitor 线程 + Flask 线程

第二阶段：前端统一（侧边栏导航）
────────────────────────────

4. 新建 index.html —— 统一界面
   布局：
   ┌──────────┬──────────────────────────┐
   │ 侧边栏    │                          │
   │          │       内容区              │
   │ 📋 任务  │   （根据选中标签动态渲染） │
   │ ⏱️ 切片  │                          │
   │ ⚙️ 设置  │                          │
   │          │                          │
   └──────────┴──────────────────────────┘

   - 侧边栏：固定宽度约60-80px（图标+文字），深色背景，高亮当前标签
   - 任务标签加载原PC助理的界面
   - 切片标签加载原切片系统的界面
   - 设置标签：通知偏好、数据导出、日志清理
   - Vue 3 CDN，用 v-if / v-show 按标签切换显示

5. 拆分 app.js
   - app.js：Vue 主应用（createApp），管理 currentTab 切换，挂载全局方法
   - tasks.js：PC助理模块的 data/methods（从原 PC助理/app.js 完整迁移）
   - slices.js：切片模块的 data/methods（从原 切片/app.js 完整迁移）
   - HTML 中按顺序加载：tasks.js → slices.js → app.js
   - app.js 使用 mixin 或 Object.assign 合并两个模块的 data/methods
