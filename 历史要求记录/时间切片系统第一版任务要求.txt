【背景】
PC智能任务助理第一、二版已完成，技术栈已验证：Python Flask + SQLite + Vue 3 CDN。
现在启动「时间切片系统」第一版开发。这是三大系统的底层数据基础设施。

项目位置：First CC/ 目录下，与 PC智能任务助理/ 平级。

【关键决策（已确认，直接执行）】
1. 独立应用：新建 时间切片系统/ 目录，独立的 Flask + 独立 SQLite
2. 数据库隔离：使用独立的 slices.db。需要读取PC助理的任务列表时，ATTACH 它的 task_assistant.db 跨库查询
3. 先做细粒度：自动后台抓取活跃窗口状态，精确到分钟
4. 切片字段（全要）：时间戳、活动描述、情绪标签（多选）、情绪强度(1-5)、精力水平(1-5)、关联任务（从PC助理读取）、自由笔记、后台设备使用记录
5. 查看界面：按日期列表 + 热力图概览（日历热力图）

【第一版范围】

■ 必须做

A. 后台窗口监控模块
- 独立脚本或 Flask 后台线程（由你推荐并说明理由）
- 每 N 秒抓取当前活跃窗口标题 + 进程名（建议用 pywin32 + psutil）
- 存入 device_usage_logs 表（时间戳精确到分钟）
- 支持优雅启停（Flask 启动时自动开始，关闭时停止）
- 讨论合理的轮询频率（建议 10-15 秒一次，然后按分钟聚合）

B. 手动切片录入
- Web 界面：包含全部字段的表单
- 情绪标签：预定义多选（焦虑/平静/兴奋/低落/烦躁/疲惫/专注/放松/其他），可用 tag 选择器
- 关联任务：从PC助理 task_assistant.db 读取，展示为下拉列表供选择
- 活动描述和自由笔记为文本输入
- 精力/情绪强度用星星或滑块

C. 按日期列表查看
- 默认显示今天的切片（自动窗口记录 + 手动切片混合展示）
- 日期选择器切换日期
- 每条切片显示所有字段
- 自动窗口记录可折叠/展开

D. 热力图概览
- 日历热力图，CSS Grid 实现就行，不需要第三方库
- 每个格子 = 一天，颜色深度 = 当天切片总数量
- 至少显示最近 3 个月
- 点击某天跳转到该天的切片列表

E. 跨库读取 PC助理任务
- 用 ATTACH DATABASE 临时挂载 task_assistant.db
- 读取 tasks 表中的任务列表供切片关联
- 路径可配置（两个应用是不同目录）

■ 不做（预留接口即可）
- 粗粒度标注（上午/下午/晚上）、情绪趋势分析（情感辅助系统的事）、时间轴视图、周视图/甘特图

【数据库设计 — slices.db】

至少需要以下表：

1. slices（手动切片记录）
   - id, timestamp（记录时间，非录入时间）, activity_desc, mood_tags（JSON数组或逗号分隔）,
     mood_intensity(1-5), energy_level(1-5), related_task_id（可空，来自task_assistant.db）,
     notes, created_at

2. device_usage_logs（自动窗口监控记录）
   - id, timestamp（精确到分钟）, window_title, process_name, duration_seconds（该分钟的活跃时长）,
     created_at

3. 可选：mood_tags 的字典表（id, name, color），方便前端渲染和后续分析

建表前给我审一下 SQL。

【技术要求】

- 跨库读取：用 sqlite3 的 ATTACH 功能，路径用相对路径 ../PC智能任务助理/task_assistant.db
- 保持弹出窗口模式（和 PC助理一致）
