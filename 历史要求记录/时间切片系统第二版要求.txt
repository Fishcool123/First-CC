【核心变更：桌面应用化】
使用 pywebview 替代浏览器弹出窗口：
- 用原生窗口嵌入现有 Flask 前端（Vue代码全部保留不动）
- 窗口关闭 → 隐藏到系统托盘，后台监控继续运行
- 右键托盘图标：显示窗口 / 退出（退出时停止监控线程）
- 原生 Windows 通知替代浏览器 Notification API（用 plyer 库）
- start.bat 改为启动 pywebview 窗口

架构变化：
  现在：Flask → 浏览器打开 localhost:xxxx
  第二版：Flask → pywebview 原生窗口（内嵌同一页面）
                 └── 系统托盘（关闭→隐藏，监控不中断）

【必须修复的 Bug】

■ Bug #1 🔴 热力图翻月偏移（app.js 约第53行）
- 原因：computed 属性 heatmapOffset 没有 setter，this.heatmapOffset = ... 静默失败
- 表现：切换到其他月份时，月初空白格子偏移错误（2月/3月/...全部错位）
- 修复：把 heatmapOffset 改为 data 属性（加 _offset），在 heatmapCells computed 或月份切换方法中更新；或者给 computed 加 get/set
- 同时删掉 app.js 第53行 computed 里的副作用赋值 this.heatmapOffset = ...

■ Bug #2 🟡 PC助理数据库文件名错误（database.py 第13行）
- 当前：'../PC智能任务助理/data.db'
- 实际：'../PC智能任务助理/task_assistant.db'
- 修复：改文件名

■ Bug #3 🟡 热力图区分度低
- 原因：device_usage_logs 数量（每天200+条）碾压手动 slices（每天3-5条）
- 表现：只要开机就是满格深色，热力图变成「你今天开电脑了吗」
- 修复：热力图只统计 slices 表的手动切片数；设备日志有无用另一个视觉标记（如格子底部小圆点表示「该日有设备数据」）

■ Bug #4 🟡 时间戳格式不统一
- monitor.py 第65行：%Y-%m-%d %H:%M（无秒）
- app.py 第117行：%Y-%m-%d %H:%M:%S（有秒）
- database.py 查询中混用 T 分隔符和空格分隔符
- 修复：统一为 YYYY-MM-DD HH:MM:SS

■ Bug #5 🟢 连续记录时时间戳不自动推进（app.js 约126-129行）
- 表现：切完第一条切片后，第二条切片的时间戳仍是同一时刻
- 修复：提交成功后自动把时间戳推进到当前时间
